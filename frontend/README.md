# Procurement Auditor — Frontend Application

Frontend web application (Tier 1) for the **G6 Automated Public Health Procurement Auditor**.

This application provides a dashboard for auditors and public health officials to:

- Authenticate securely using JWT
- View procurement contracts
- Submit new contracts for automated risk analysis
- View fraud alerts generated by the audit engine
- Compare awarded prices against KEMSA benchmark prices
- Monitor suspicious procurement activities

Built with React + TypeScript + Vite and designed as an enterprise SaaS dashboard.

---

## Stack

- React 18
- TypeScript
- Vite
- React Router
- Axios (API communication)
- Tailwind CSS
- JWT authentication
- Modern component-based architecture

---

## Requirements

Before running the frontend, ensure you have:

- Node.js 18+
- npm 9+
- Backend API running locally

Backend should be available at:
http://localhost:5000


Frontend runs on:
http://localhost:5173

# Setup

## 1. Install dependencies

Navigate to the frontend folder:

```bash
cd frontend

Install packages:
npm install

2. Configure environment variables

Create a .env file inside the frontend directory:
frontend/.env

Add:
VITE_API_URL=http://localhost:5000/api

This URL is used by Axios to communicate with the backend API.

Running the Application
Development mode

Start the Vite development server:
npm run dev

The application will start at:
http://localhost:5173

Production build

Create an optimized production build:
npm run build

Application Flow
Authentication

Users authenticate through:
POST /api/auth/register
POST /api/auth/login

After successful login:

Backend returns JWT token
Frontend stores authentication state
Token is attached to protected API requests

Protected requests include:
Authorization: Bearer <token>
